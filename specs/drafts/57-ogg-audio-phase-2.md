# Plan #57: Audio phase 2 — OGG container (page parser + Vorbis/Opus comments)

## Problem
XIFty supports FLAC + Vorbis comments (issue #55, merged) but does not yet recognise OGG-encapsulated audio. Task #57 asks for an `OggS`-framed page parser that can surface Vorbis and Opus logical streams, decode their identification headers (sample rate, channels, codec), reuse `xifty-meta-vorbis-comment` for tag decoding, and compute duration from the last-page granule position. Detection, probe, and extract must all see OGG, and a new `ogg` container capability must be declared. No FFI shape change.

## Approach
Mirror the FLAC landing in issue #55 one-to-one:
- New container crate `crates/xifty-container-ogg/` with a minimal `parse_bytes` that walks the OGG page chain (magic `OggS`), reassembles packets for the primary logical stream, identifies Vorbis vs Opus from the first packet, decodes ident-header fields, extracts the Vorbis-comment packet payload for `xifty-meta-vorbis-comment`, and surfaces the last-page granule position for duration.
- Mirror container surface shape of `FlacContainer` (`nodes`, `issues`, `stream_info`-equivalent, `vorbis_comment` payload view) so the CLI wiring is copy-of-FLAC.
- `xifty-detect` adds an `OggS` magic branch returning new `Format::Ogg` variant.
- `xifty-cli` adds a new `Format::Ogg` arm in both `probe_source` and `extract_source`, constructing `AudioSampleRate` / `AudioChannels` / `AudioBitDepth` (omitted for Opus — 48 kHz fixed decoded rate; spec's "input sample rate" is surfaced separately as `OpusInputSampleRate` note-only) / `DurationSeconds` / `AudioCodec` metadata entries in a new `ogg` namespace, then feeds the Vorbis-comment block through `decode_vorbis_comment_payload` with `container: "ogg"`.
- Policy already normalizes `AudioSampleRate`/`AudioChannels`/`AudioBitDepth`/`DurationSeconds`/`AudioCodec` regardless of source namespace (`crates/xifty-policy/src/lib.rs:181-238`), so no policy change is required.
- Fixture generator adds synthetic `happy.ogg` (Vorbis) and `happy.opus` (Opus) files built from Python byte-level; tests mirror the FLAC snapshot pattern.

## Step-by-step
1. **Scaffold the container crate.** Create `crates/xifty-container-ogg/` with `Cargo.toml` mirroring the FLAC crate and an empty `src/lib.rs`. Add the member to the top-level `Cargo.toml`. Verifiable: `cargo check -p xifty-container-ogg` succeeds with a placeholder module.
2. **Implement the page parser.** In `crates/xifty-container-ogg/src/lib.rs` write a `parse_bytes(bytes, base_offset)` that walks the chain of OGG pages: each page begins with `OggS`, version `0`, then `header_type`, `granule_position` (i64 LE), `bitstream_serial_number` (u32 LE), `page_sequence` (u32 LE), `crc` (u32 LE), `page_segments` (u8), and `segment_table` (u8 × page_segments). Payload length = sum of segment lengths. Surface each page as a `ContainerNode { kind: "page" }` child of the `"ogg"` root node, record `Issue`s for truncation / bad magic / unexpected version. Public return type `OggContainer { nodes, issues, first_codec: Option<OggCodec>, vorbis_ident: Option<VorbisIdent>, opus_ident: Option<OpusIdent>, vorbis_comment: Option<VorbisCommentBlock>, granule_last: Option<i64>, serial: Option<u32> }`. Verifiable: unit tests in the crate cover (a) two pages reassembling into one packet, (b) continuation-flag fragmentation across segment size 255, (c) truncated trailing segment table surfaces `ogg_page_truncated` issue, (d) `OggS` magic mismatch returns `XiftyError::Parse`.
3. **Reassemble packets and identify codec.** Build an inner packet iterator from segments (segments of length < 255 terminate a packet; runs of 255 continue). Inspect the first reassembled packet: if it starts with `\x01vorbis` it is Vorbis — decode the Vorbis ident header (version u32 LE, channels u8, sample_rate u32 LE, then bitrate fields; see Vorbis I §4.2). If it starts with `OpusHead` it is Opus — decode version u8, channels u8, pre_skip u16 LE, input_sample_rate u32 LE, output_gain i16 LE, channel_map u8. The second packet is the comment-header packet: for Vorbis it has the `\x03vorbis` prefix with the Vorbis-comment payload immediately following; for Opus it has the `OpusTags` prefix then the same payload format. Store the payload slice start/end as `VorbisCommentBlock { offset_start, offset_end, data_offset, data_length }` (matching the FLAC crate shape at `crates/xifty-container-flac/src/lib.rs:33-39`). Verifiable: unit tests decode Vorbis and Opus ident/comment headers from hand-built pages and assert `sample_rate`, `channels`, `codec` variant, and comment payload offsets.
4. **Track duration via last-page granule.** While iterating pages, keep the last page's `granule_position`. For Vorbis, duration_seconds = granule / ident.sample_rate. For Opus, duration_seconds = (granule - pre_skip) / 48000 (Opus decoded rate is always 48 kHz per RFC 7845 §4). Surface `None` when granule is 0 or negative (raw/invalid) and emit an `ogg_granule_missing` warning. Verifiable: unit test builds a 2-page stream with granule = sample_rate and asserts duration ≈ 1.0 s for Vorbis; separate test asserts Opus duration computation.
5. **Add `Format::Ogg` and detection.** Extend `crates/xifty-core/src/lib.rs` with `Format::Ogg` and `"ogg"` in `as_str`. Extend `crates/xifty-detect/src/lib.rs` to return `Format::Ogg` when `bytes[0..4] == b"OggS"`, and add `ogg` file cases to the `detects_formats` test. Verifiable: `cargo test -p xifty-detect` passes the extended test.
6. **Wire CLI probe/extract arms.** In `crates/xifty-cli/src/lib.rs` add `use xifty_container_ogg::{OggContainer, OggCodec, parse as parse_ogg};`. Add `Format::Ogg =>` arm in `probe_source` returning `("ogg", parsed.nodes, parsed.issues)`. Add `Format::Ogg =>` arm in `extract_source` that calls a new `ogg_entries(&ogg, source.bytes(), &mut issues)` helper that emits (a) `AudioSampleRate`, `AudioChannels`, (Vorbis-only) `AudioBitDepth` = 16 literal when not present / omitted, `DurationSeconds`, `AudioCodec` ("vorbis" | "opus") in namespace `"ogg"` with provenance pointing at the ident header's byte range, and (b) runs the Vorbis-comment payload through `decode_vorbis_comment_payload` with `container: "ogg"`. Emit `ogg_vorbis_comment_decode_empty` issue if the payload decodes to zero entries (mirroring `flac_entries` at `crates/xifty-cli/src/lib.rs:1043-1073`). Verifiable: `cargo check -p xifty-cli` succeeds; manual `cargo run -p xifty-cli -- probe fixtures/minimal/happy.ogg` reports codec + rate.
7. **Extend the fixture generator.** Add `build_ogg_vorbis(...)` and `build_ogg_opus(...)` functions in `tools/generate_fixtures.py` that emit spec-compliant OGG page chains containing: (i) ident header page (1 packet), (ii) comment+setup-header packet(s) as a separate page — for Vorbis the comment packet carries `TITLE=XIFty Track`, `ARTIST=XIFty Artist`, `ALBUM=XIFty Album` with vendor `xifty-test`; for Opus the `OpusTags` packet carries the same Vorbis-comment payload. Use serial `0x12345678`, and set the last page's granule_position so duration rounds to `1.0`. Register `"happy.ogg"` and `"happy.opus"` near `crates/generate_fixtures.py:1009-1012`. CRC can be computed with the standard OGG CRC polynomial `0x04C11DB7` (implement once in the script). Verifiable: `python tools/generate_fixtures.py` regenerates fixtures; the new files exist in `fixtures/minimal/`.
8. **Add CLI snapshot + behaviour tests.** In `crates/xifty-cli/tests/cli_contract.rs` append test functions mirroring `probe_snapshot_happy_flac` / `extract_snapshot_happy_flac_normalized` / `flac_normalization_includes_audio_fields` / `flac_surfaces_vorbis_comment_tags` (lines 2132-2214) for `happy.ogg` and `happy.opus`. Assert: normalized `audio.sample_rate`, `audio.channels`, `duration`, `codec.audio`; interpreted `vorbis_comment` Title/Artist/Album round-trip; extracted codec is `vorbis` or `opus`. Verifiable: `cargo test -p xifty-cli --test cli_contract` after `cargo insta review`.
9. **Capability declaration.** Update `CAPABILITIES.json` under `containers` with the `ogg` entry and under `namespaces` with an `ogg` section listing `AudioSampleRate`, `AudioChannels`, (Vorbis-only) `AudioBitDepth`, `DurationSeconds`, `AudioCodec`. Verifiable: `python tools/validate_schema_examples.py` (if present) still passes and the JSON is valid.
10. **Docstring cleanup.** Replace the pointer comment in `crates/xifty-meta-vorbis-comment/src/lib.rs:6-9` with a fixed reference ("OGG passes `\"ogg\"` as the container"). Verifiable: no behaviour change; rustdoc builds.
11. **Run the validation gates locally.** Run the commands listed in §Validation below. Address any insta drift through `cargo insta review` — never blanket-accept (per SRS §3 and guardrails).

## Risks
- **CRC computation in Python fixture generator** — OGG pages include a CRC that decoders MAY validate. The container parser here is tolerant (does not require CRC validity) but a future hardening might. Plan documents a CRC helper in `generate_fixtures.py` to keep fixtures spec-clean; if time-boxed out, the parser must explicitly tolerate `crc == 0` pages and the tolerance must be covered by a unit test.
- **Opus "input sample rate" vs "decoded sample rate"** — Opus always decodes to 48 kHz. The ident header's `input_sample_rate` is metadata about the source encoder's rate and is commonly 48000 but not guaranteed. We surface `AudioSampleRate = 48000` (decoded) to keep the normalized `audio.sample_rate` semantically consistent with how users query audio files; the raw `input_sample_rate` is preserved as an `OpusInputSampleRate` entry in the `ogg` namespace with a note for provenance fidelity. Decision captured in the container crate doc comment.
- **Multiplexed OGG streams** — OGG can chain multiple logical streams (Vorbis+Skeleton, OGM, OGG Theora+Vorbis). Phase 2 scope is single-logical-stream audio: parser selects the first logical stream serial it sees, ignores others, and emits an `ogg_multiplexed_streams_unsupported` info-level issue when additional serials are present. Full chaining support is a future task.
- **Bitdepth** — Vorbis and Opus do not define a "bits per sample" in the ident headers. We emit `AudioBitDepth = 16` only for Vorbis with a note `"default assumption; vorbis ident header does not encode bits_per_sample"`, and omit `AudioBitDepth` for Opus. Normalized `audio.bit_depth` will accordingly be absent for `.opus` — the existing FLAC snapshot path expects presence (`crates/xifty-cli/tests/cli_contract.rs:2163-2166`) so the new Opus behavioural test must assert *absence* explicitly to avoid regression.
- **Snapshot drift** — any touch of the CLI arms may shift existing snapshots indirectly. Policy-level changes are not intended; if any pre-existing snapshot diffs, the reviewer must triage via `cargo insta review` (SRS §3.6, §4).
